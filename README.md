# Ulama Circle — Day 1 scaffold

This is not a runnable Flutter project by itself — it's the `lib/`, config,
and folder structure to drop into one. I don't have the Flutter SDK available
in my build environment, so these steps need to run on your machine.

## Setup steps (Day 1 of the plan)

1. **Create the actual Flutter project** (if you haven't already):
   ```
   flutter create ulama_circle
   ```

2. **Copy these files in**, overwriting the generated `lib/`, `pubspec.yaml`,
   and adding `firestore.rules` and `README.md` at the project root.

3. **Get packages:**
   ```
   flutter pub get
   ```
   Note: the `algolia` package version in `pubspec.yaml` is a placeholder —
   check pub.dev for the current latest version before running this, since I
   can't verify it live from here.

4. **Create the Firebase project** at console.firebase.google.com, then connect it:
   ```
   dart pub global activate flutterfire_cli
   flutterfire configure
   ```
   This generates `lib/firebase_options.dart`. Once it exists, uncomment the
   two related lines in `lib/main.dart` (the import and the
   `Firebase.initializeApp(options: ...)` call) and remove the bare
   `await Firebase.initializeApp();` call above it.

5. **Upgrade the Firebase project to the Blaze plan** — required later for
   Cloud Functions and the Algolia extension, may as well do it now.

6. **Deploy the security rules:**
   ```
   firebase deploy --only firestore:rules
   ```
   (requires `firebase-tools` installed and `firebase init` run once first)

7. **Run it:**
   ```
   flutter run
   ```
   You should see the dark charcoal-and-gold splash screen. That's the Day 1
   checkpoint — project boots, Firebase connects, Hive opens without errors.

## About the Hive adapter

`lib/models/downloaded_lecture_model.g.dart` is normally auto-generated by
`build_runner`. I hand-wrote it to match the expected output so the project
compiles immediately without a codegen step. If you add or change fields on
`DownloadedLecture` later, delete that file and run:
```
dart run build_runner build --delete-conflicting-outputs
```
to get a correctly regenerated version.

## What's included in this Day-1 drop

- Full folder structure matching the spec (`lib/core`, `services`, `models`,
  `providers`, `screens/*`, `widgets`, `routes`)
- `pubspec.yaml` with every package from the spec
- Core models: `LectureModel`, `SheikhModel`, `CategoryModel`, `CommentModel`,
  `PlaylistModel`, `DownloadedLecture` (Hive)
- `AppTheme` / `AppColors` — the charcoal-and-gold palette
- `AppConstants` — every collection/box/topic name in one place
- `firestore.rules` — matches section 3 of the spec (public read on content,
  admin-only writes, owner-only user data)
- `main.dart` / `app.dart` / a placeholder `splash_screen.dart` — enough for
  the app to actually boot and prove the wiring works
